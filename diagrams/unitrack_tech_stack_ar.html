⚛️ React.js
المكتبة الأساسية للواجهة
 v18 المكتبة الأساسية لبناء تطبيق الصفحة الواحدة (SPA) الديناميكي. تعتمد على البنية القائمة على المكونات لبناء عناصر واجهة قابلة لإعادة الاستخدام. تدير الحالات المعقدة لأربعة أدوار مستخدمين.
📦 المكتبات المضمّنة
React Router DOM v6 — تنقل سلس بين صفحات لوحة التحكم مع حراسة مسارات قائمة على الأدوار ومسارات محمية.
Axios — عميل HTTP قائم على Promises لمعالجة جميع طلبات API إلى الخلفية. يدعم Interceptors لإرفاق رمز JWT تلقائياً ومعالجة الأخطاء.
Lucide React / Heroicons — مكتبات أيقونات SVG احترافية توفر أكثر من 1000 أيقونة للقوائم والإجراءات وحالات الإشعارات.
Framer Motion اختياري — مكتبة تحريك تصريحية لتأثيرات واجهة سلسة تشمل انتقالات الصفحات وظهور النوافذ وحركات عناصر القوائم.
🎨 Tailwind CSS
التصميم والتنسيق
 v3 إطار CSS قائم على الأدوات المساعدة للحصول على تصميم واجهة عصري ومتجاوب ومتسق. يُلغي الحاجة لملفات CSS مخصصة ويسمح بتنسيق سريع مباشرةً في JSX.
📦 المكتبات المضمّنة
Headless UI — مكونات واجهة بدون تنسيق مسبق وقابلة للوصول بالكامل (Modal، Dropdown، Tabs) مصممة للتكامل مع Tailwind CSS.
Tailwind Merge (twMerge) — أداة لدمج أسماء فئات Tailwind CSS بذكاء دون تعارضات أنماط التنسيق، خاصةً في خصائص المكونات الديناميكية.
clsx — أداة خفيفة لضم أسماء فئات CSS بشكل شرطي، مما يوفّر إدارة أنظف للفئات الديناميكية في مكونات React.
📜 JavaScript (ES6+)
لغة منطق العميل
 ES2022 لغة البرمجة الأساسية لجميع المنطق التفاعلي من جانب العميل. تُطبق async/await لاستدعاء API، والدوال السهمية، وتفكيك المتغيرات، والوحدات. تُشغّل معالجة النماذج وتحويل البيانات وتحديثات الحالة وتكامل Web Speech API للأوامر الصوتية.
🔀 React Router DOM
التوجيه من جانب العميل
 v6 يتولى التنقل بين الصفحات وحماية المسارات عبر أكثر من 30 صفحة. يُنفّذ حراسة المسارات القائمة على الأدوار للتحقق من أن كل مستخدم (مدير / أستاذ / معيد / طالب) يصل فقط لصفحاته المخصصة.
⚡ Vite
أداة البناء والخادم
 v5 أداة بناء وخادم تطوير فائقة السرعة. توفر Hot Module Replacement (HMR) فورياً وبدايات سريعة وحزماً إنتاجية محسّنة. تُنشر عبر Vercel للتسليم المستمر.

الخلفية وجانب الخادم (Node.js / Laravel)
معالجة الخادم ومنطق الأعمال والمصادقة ومعالجة الملفات
الأداة / الإطار اللغة الدور والغرض والمكتبات المضمّنة
🟢 Node.js / Laravel
مكدس الخلفية الأساسي
 JS / PHP Node.js (بيئة تشغيل JavaScript) أو Laravel (إطار PHP MVC) لبناء جميع نقاط نهاية RESTful API، ومعالجة منطق الأعمال، وطبقات Middleware، ومعمارية MVC.
📦 المكتبات المضمّنة
Bcrypt / Hash — مكتبة معيارية لـتجزئة كلمات المرور بسرية. تُجزّأ جميع كلمات مرور المستخدمين قبل التخزين باستخدام خوارزمية bcrypt مع جولات التمليح التكيفية، مما يجعلها لا رجعية ومقاومة لهجمات القوة الغاشمة.
JWT (JSON Web Tokens) — آلية مصادقة قائمة على الرموز الخالية من الحالة. عند تسجيل الدخول بنجاح، يُصدر JWT موقّع ويُرفق بكل طلب لاحق لـإدارة الجلسة بأمان دون جلسات خادم.
Multer / Storage Facade (Laravel) — وسيط لمعالجة رفع الملفات متعددة الأجزاء. يدير تسليمات ملفات المهام (PDF، ZIP)، ويتحقق من أنواع MIME، ويفرض حدود الحجم، ويحفظ الملفات في نظام الملفات أو التخزين السحابي.
🐘 PHP
لغة الخلفية الأساسية
 v8.x اللغة الأساسية لـمعالجة الخلفية في مكدس Laravel. تتعامل مع دورات HTTP/Middleware ومنطق المصادقة واستعلامات ORM عبر Eloquent.
🐍 FastAPI
نشر الخلفية الحالي (Python)
 Python إطار ويب Python يُشغّل النشر الحالي (أكثر من 50 مساراً). يتعامل مع الإشعارات ورفع الملفات وميزات الذكاء الاصطناعي عبر Uvicorn ASGI على Railway.
📦 المكتبات المضمّنة
Python-Jose (JWT) — توليد رموز JWT والتحقق منها للمصادقة الآمنة عبر جميع طلبات API.
Passlib / Bcrypt — تجزئة كلمات المرور والتحقق منها باستخدام خوارزمية bcrypt مع جولات تمليح قابلة للتكوين.
Python-Multipart — يعالج رفع الملفات متعددة الأجزاء لنقاط نهاية تسليم مهام الطلاب.
google-generativeai — SDK ذكاء Google Gemini للمساعد المحادث وتلخيص الاجتماعات ونقاط صحة الفريق.

🗄️ MySQL
محرك قاعدة البيانات (معيار)
 v8.x نظام إدارة قواعد البيانات العلائقية (RDBMS) المعتمد للنشر الإنتاجي. يخزن جميع حسابات المستخدمين وبيانات المشاريع وتسليمات المهام والدرجات مع امتثال ACID الكامل.
📦 المكتبات المضمّنة
Sequelize ORM (Node.js) — ORM قائم على Promises لـ Node.js يتيح التفاعل مع MySQL باستخدام كائنات JavaScript بدلاً من SQL الخام. يدعم تعريف النماذج والارتباطات (hasMany، belongsTo) والترحيلات وبناء الاستعلامات.
Eloquent ORM (Laravel/PHP) — ORM المدمج في Laravel القائم على Active Record. يربط جداول قاعدة البيانات بفئات PHP مع باني استعلامات سلس وأساليب العلاقات وطوابع زمنية تلقائية.
MySQL2 (Node.js driver) — مشغّل MySQL سريع وحديث بدعم Promise يُستخدم كموصّل قاعدة بيانات أساسي لـ Sequelize في بيئات Node.js.
🪶 SQLite
قاعدة بيانات مضمنة (النشر الحالي)
 v3 قاعدة بيانات SQL خفيفة الوزن وبلا خادم مستخدمة في النشر الحالي على سحابة Railway. تخزن نفس المخطط العلائقي كـ MySQL (13 جدولاً) بلا تكوين إضافي. مثالية لبيئات الاستضافة السحابية.
🔗 SQLAlchemy ORM
رابط بيانات Python (FastAPI)
 v2 يُعرِّف نماذج قاعدة البيانات كفئات Python. يتجرد من SQL الخام، ويعالج العلاقات، ويُنشئ الجداول الـ 13 تلقائياً عبر Base.metadata.create_all() عند بدء الخادم.
📐 نمذجة ERD
تصميم المخطط وتحليله
 أداة تصميم تُستخدم في تصميم مخطط قاعدة البيانات العلائقية قبل التنفيذ. توثّق مخططات ERD بـ Crow's Foot جميع الجداول والمفاتيح وقواعد الأساسية (1:N) لضمان سلامة البيانات.

💻 Visual Studio Code
بيئة التطوير المتكاملة (IDE)
 IDE بيئة التطوير المتكاملة الأساسية لجميع تطوير الواجهة الأمامية والخلفية. تُستخدم مع إضافات ESLint وPrettier وPython وTailwind CSS IntelliSense وGitLens لجودة الكود والتنسيق وإدارة المستودع.
📮 Postman
اختبار API
 اختبار يُستخدم على نطاق واسع لـاختبار نقاط نهاية API وتصحيحها وتوثيقها. تم اختبار جميع نقاط النهاية الـ 50+ عبر مجموعات Postman قبل تكامل الواجهة الأمامية. يتحقق من تنسيقات الطلب/الاستجابة ورموز المصادقة وسلوكيات معالجة الأخطاء.
🖥️ XAMPP / Laragon
بيئة الخادم المحلي
 تطوير محلي يوفر بيئة خادم تطوير محلية مع Apache وMySQL وPHP مُهيَّأة مسبقاً. XAMPP يوفر دعماً متعدد المنصات بينما Laragon يقدم بديلاً أخف وأسرع لـ Windows مع مضيفين افتراضيين تلقائيين وإدارة سريعة لقاعدة البيانات.
🌿 Git
نظام التحكم في الإصدارات
 VCS نظام تحكم في الإصدارات الموزّع لتتبع جميع تغييرات الكود المصدري طوال دورة حياة المشروع. يُمكّن استراتيجيات التفريع وسجل الالتزامات وإمكانية التراجع. يتعاون جميع أعضاء الفريق عبر سير عمل Git.
🐙 GitHub
استضافة الكود المصدري
 VCS منصة سحابية لـاستضافة الكود المصدري والتطوير التعاوني. يخزن مستودع UniTrack AI جميع كود الواجهة الأمامية والخلفية. يعمل أيضاً كمُشغّل للنشر على Vercel (نشر تلقائي للواجهة عند الـ push) وRailway (نشر مستمر للخلفية).

🔷 ERD (Crow's Foot)
نمذجة البيانات
 UML / DM مخطط العلاقات بين الكيانات باستخدام رموز Crow's Foot يوثّق جميع جداول قاعدة البيانات (users, roles, projects, tasks, notifications) والمفاتيح الأساسية/الخارجية وقواعد الأساسية (1:N). ضروري لضمان سلامة البيانات في المخطط العلائقي.
👤 مخطط حالات الاستخدام
تحليل المتطلبات
 UML يلتقط جميع المتطلبات الوظيفية وتفاعلات الممثلين مع النظام. يوثق 8 حالات استخدام عبر 3 ممثلين (طالب، مشرف، مدير) تشمل المصادقة ورفع المهام وتسليم الدرجات وإدارة المستخدمين وتوليد التقارير.
📊 مخطط النشاط
نمذجة سير العمل
 UML يُنمذج تدفقات العمليات خطوة بخطوة باستخدام مسارات سباحة (swimlanes) لكل ممثل. يوثق سير عمل تسليم المهمة الكامل من الرفع عبر التحقق ومراجعة المشرف وقرار الموافقة/الرفض والتقييم النهائي.
📨 مخطط التسلسل
نمذجة التفاعل
 UML يوضح تبادل الرسائل المرتّب زمنياً بين مكونات النظام. يُظهر تسلسل تسليم المهمة التفاعلاتِ بين واجهة الطالب ووحدة تحكم API وخدمة المهام وقاعدة البيانات وخدمة الإشعارات.
🔶 مخطط الفئات
البنية البرمجية OOP
 UML يحدد البنية الموجّهة للكائنات للنظام: فئة User الأساسية مع فئتي Student وSupervisor وAdmin الفرعية (وراثة)، إضافةً إلى Project وTask بسماتها وأساليبها. يُرشد تعريفات النماذج في الخلفية.
🌊 DFD (المستوى 0 و1)
تحليل تدفق البيانات
 DFD مخططات تدفق البيانات (DFD) توثّق كيفية تحرك البيانات عبر النظام. المستوى 0 (السياق) يُظهر النظام كعملية واحدة مع 3 كيانات خارجية. المستوى 1 يُفصّل 4 عمليات: المصادقة وإدارة المشاريع وتسليم المهام والتقييم — لكل منها مخازن بيانات وتدفقاتها.
🖊️ Lucidchart / Draw.io
أداة رسم المخططات
 أداة تصميم أدوات رسم مخططات احترافية قائمة على الويب تُستخدم في إنشاء جميع رسومات UML وDFD والمعمارية والمخططات الانسيابية. تدعم تدوين UML 2.5 المعياري والتصدير إلى SVG/PDF والتحرير التعاوني للفرق.
🏗️ معمارية 3 طبقات
معمارية النظام
 معمارية يوثّق معمارية النظام ثلاثية الطبقات: طبقة العرض (React.js، المتصفح) وطبقة التطبيق (FastAPI، Railway) وطبقة البيانات (SQLite). يؤكد الفصل الواضح بين المسؤوليات ويحدد بروتوكولات الاتصال (REST API عبر HTTPS وSQLAlchemy ORM).
🚀 مخطط النشر
تصميم البنية التحتية
 معمارية يُعيّن طوبولوجيا النشر الفعلية: متصفح العميل (React SPA) ← شبكة Vercel (ملفات ثابتة، HTTPS) ← خادم تطبيق Railway (FastAPI + Python) ← ملف SQLite مع اتصال خارجي بـ Google Gemini AI API.
