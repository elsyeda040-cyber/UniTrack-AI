Diagram 06
AI-Powered Features (Google Gemini Integration)
UniTrack AI leverages Google Gemini 2.5 Flash for intelligent academic assistance across all user roles.
🤖
AI Copilot Chat
Context-aware AI assistant available to students, professors, and assistants. Responds in Arabic. Aware of Cairo time (UTC+2). Endpoint: POST /ai/chat
💡
Team Health Score
Analyzes last 50 chat messages + task statuses to generate a Team Health Score (0-100) with collaboration, progress, and morale metrics. Endpoint: POST /teams/{id}/insights
📋
Chat Summarizer
Generates a concise Arabic bullet-point summary of the team's chat history. Highlights key discussions, decisions made, and remaining tasks. Endpoint: POST /teams/{id}/chat-summary
📚
Syllabus Scraper
Takes a syllabus or project description and auto-suggests 5-8 learning resources (videos, docs, articles) in structured JSON. Endpoint: POST /ai/scrape-syllabus
🎤
Presentation Coach
AI-powered feedback on student presentations. Evaluates delivery, content structure, and body language. Frontend page: /student/presentation-coach
💻
Code Mentor
Helps students with code reviews, debugging, and best practices. Provides contextual code suggestions and explanations. Frontend page: /student/code-mentor
🎙️
Voice Command System
Global voice control using Web Speech API. Navigate between pages, trigger actions using voice. Available to all users via floating GlobalVoiceControl component.
🏢
Meeting AI Copilot
In the Meeting Assistant, AI provides live transcription, meeting summaries, and action-item extraction. Available to professors as host with full admin controls.
Diagram 07
Gamification & Motivation System
A complete reward system to motivate students through badges, leaderboards, peer reviews, and credit scoring.
🏅
Badge System
Students earn badges automatically based on performance. E.g. "The Finisher" badge awarded when team completes 3+ tasks. Badges stored in user_badges table. Email notification sent on award.
🏆
Leaderboard
Top 10 students ranked by formula:
(Completed Tasks × 50) + (Avg Score × 5) + (Badges × 100)
Updated dynamically via GET /leaderboard. Visible to all roles.
💰
Credits System
Each student has a credits balance (default 100). Credits can be used in Community Market interactions. Stored in users.credits column. Used in Help Request bounties.
⭐
Peer Review
Students can rate each other (1-5 stars) with comments within their team. One review per reviewer-reviewee pair per team. Updates on resubmission. Email notification sent to reviewee.
🎯
Risk Simulator
Students can simulate project risk scenarios to understand the impact of delays, scope changes, and resource issues on project success. Accessible via RiskSimulator page.
🛒
Community Market
Marketplace where students can buy/sell services, resources, or help using credits. Foster a collaborative academic economy within the platform. Route: /student/market

Diagram 10
Notification & Email System Flow
Automated in-app notifications and email alerts triggered by key events across the platform.
📧 Email Triggers:
🆕 New User Created
Admin creates user → Welcome email sent with role and team info
📊 Task Scored
Professor grades task → Email sent to all team members with feedback notification
🏅 Badge Awarded
Student earns badge (e.g. The Finisher) → Congratulations email sent immediately
📅 New Event
Professor creates meeting/milestone → Email to professor + all students in team
⭐ Peer Review
Student submits peer review → Email sent to the reviewee with reviewer name
💬 New Chat Message
Member sends message → In-app notification created for all other team members (not email)
🔔 In-App Notification Flow:
Trigger Event
→
Backend creates Notification record
→
GET /users/{id}/notifications
→
Bell icon shows unread count
→
Notification page / mark as read
