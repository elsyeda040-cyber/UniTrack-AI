Frontend Development (React.js Environment)
Client-side technologies, UI libraries, and dependencies
Tool / Framework Version / Tag Role, Purpose & Included Libraries
⚛️ React.js
Core UI Library
 v18 Core library for building a dynamic Single Page Application (SPA). Uses component-based architecture to build reusable UI elements. Manages complex UI states including dashboards, task lists, notifications, and real-time chat across four user roles.
📦 Libraries Included
React Router DOM v6 — Seamless client-side navigation between dashboard pages with role-based route guards and protected routes.
Axios — Promise-based HTTP client for handling all API requests to the backend. Supports interceptors for automatic JWT token attachment and error handling.
Lucide React / Heroicons — Professional, consistent SVG icon libraries providing 1,000+ system icons for menus, actions, statuses, and notifications.
Framer Motion Optional — Declarative animation library for smooth UI micro-animations including page transitions, modal entrances, and list item reveals.
🎨 Tailwind CSS
Styling & Design Framework
 v3 Utility-first CSS framework for modern, responsive, and consistent UI styling. Enables rapid design iteration directly in JSX without custom CSS files. Used for all layout, spacing, typography, colors, and responsive grids.
📦 Libraries Included
Headless UI — Completely unstyled, fully accessible UI component primitives (Modals, Dropdowns, Tabs) designed to integrate with Tailwind CSS.
Tailwind Merge (twMerge) — Utility for intelligently merging Tailwind CSS class names without style conflicts, especially in dynamic component props.
clsx — Lightweight utility for conditionally joining CSS class names, enabling clean dynamic class management in React components.
📜 JavaScript (ES6+)
Client-Side Logic
 ES2022 Core programming language for all client-side logic and interactivity. Implements async/await for API calls, arrow functions, destructuring, and modules. Powers form handling, data transformation, state updates, and the Web Speech API integration for voice commands.
⚡ Vite
Build Tool & Dev Server
 v5 Ultra-fast frontend build tool and development server. Provides instant Hot Module Replacement (HMR), fast cold starts, and optimized production bundles. Deployed via Vercel for continuous delivery.
2
Backend & Server Side (Node.js / Laravel / FastAPI)
Server-side processing, business logic, authentication, and file handling libraries
Tool / Framework Language Role, Purpose & Included Libraries
🟢 Node.js / Laravel
Primary Backend Stack
 JS / PHP Node.js (JavaScript runtime) or Laravel (PHP MVC framework) used to build all RESTful API endpoints, handle server-side business logic, middleware, and MVC architecture for the platform.
📦 Libraries Included
Bcrypt / Hash — Industry-standard library for secure password hashing. All user passwords are hashed before storage using bcrypt's adaptive salt rounds, making them irreversible and brute-force resistant.
JWT (JSON Web Tokens) — Stateless token-based authentication mechanism. Upon successful login, a signed JWT is issued and attached to every subsequent request header for secure session management without server-side sessions.
Multer / Storage Facade (Laravel) — Middleware for handling multipart/form-data file uploads. Manages student task file submissions (PDFs, ZIPs, documents), validates MIME types, enforces file size limits, and stores files to the server filesystem or cloud storage.
🐘 PHP
Core Backend Language
 v8.x Core language for server-side processing in the Laravel stack. Handles HTTP request/response cycles, middleware pipeline, authentication logic, ORM object mapping, and server-side business rule enforcement.
🐍 FastAPI
Active Deployment API (Python)
 Python High-performance Python web framework powering the current live deployment (50+ REST routes). Handles all platform features including AI integration, notifications, gamification, and file management via Uvicorn ASGI on Railway cloud.
📦 Libraries Included
Python-Jose (JWT) — JWT token generation and verification for secure authentication across all API requests.
Passlib / Bcrypt — Secure password hashing and verification using bcrypt algorithm with configurable salt rounds.
Python-Multipart — Handles multipart file uploads for student task submission endpoints.
google-generativeai — Google Gemini AI SDK for conversational assistant, meeting summarization, and team health scoring features.
3
Database & Connectivity
Database engine, ORM libraries, schema design, and data access layers
Tool / Technology Version / Tag Role, Purpose & Included Libraries
🗄️ MySQL
Database Engine (Standard)
 v8.x Industry-standard open-source Relational Database Management System (RDBMS) recommended for production deployments. Stores all user accounts, project metadata, task submissions, grades, feedback, notifications, and meeting records with full ACID compliance and referential integrity.
📦 Libraries Included
Sequelize ORM (Node.js) — Promise-based ORM for Node.js enabling interaction with MySQL using JavaScript objects instead of raw SQL. Supports model definition, associations (hasMany, belongsTo), migrations, and query building.
Eloquent ORM (Laravel/PHP) — Laravel's built-in Active Record ORM. Maps database tables to PHP model classes with fluent query builder, relationship methods, and automatic timestamps for all project entities.
MySQL2 (Node.js driver) — Fast, modern MySQL driver with Promise support used as the underlying database connector for Sequelize in Node.js environments.
🪶 SQLite
Embedded Database (Active Deployment)
 v3 Lightweight, serverless SQL database used in the current active deployment on Railway cloud. Stores the same relational schema as MySQL (13 tables) with zero configuration overhead. Ideal for cloud hosting environments without a dedicated database server.
🔗 SQLAlchemy ORM
Python Database Mapper (FastAPI)
 v2 Python ORM defining database models as Python classes. Abstracts raw SQL, handles entity relationships (one-to-many, many-to-many via association tables), and auto-generates all 13 tables via Base.metadata.create_all() on server startup.
📐 ERD Modeling
Schema Design & Analysis
 Design Tool Used for designing the relational database schema before implementation. ERDs with Crow's Foot Notation document all tables, primary/foreign keys, and 1:N cardinality rules to ensure data integrity and normalization compliance.
4
Development & Productivity Tools
IDE, testing, local environment, and version control toolchain
Tool Category Role & Purpose
💻 Visual Studio Code
Primary IDE
 IDE The primary Integrated Development Environment (IDE) for all frontend and backend development. Used with extensions including ESLint, Prettier, Python, Tailwind CSS IntelliSense, and GitLens for code quality, formatting, and repository management.
📮 Postman
API Testing Tool
 Testing Used extensively for API endpoint testing, debugging, and documentation. All 50+ REST API endpoints were tested through Postman collections before frontend integration. Validates request/response formats, authentication tokens, and error handling behaviors.
🖥️ XAMPP / Laragon
Local Server Environment
 Local Dev Provides a local development server environment with Apache, MySQL, and PHP pre-configured. XAMPP offers cross-platform support while Laragon provides a lighter, faster alternative for Windows with automatic virtual hosts and quick database management.
🌿 Git
Version Control System
 VCS Distributed version control system for tracking all source code changes throughout the project lifecycle. Enables branching strategies, commit history, and rollback capabilities. All team members collaborate through Git workflows (feature branches, pull requests, merging).
🐙 GitHub
Remote Repository
 VCS Cloud-based platform for source code hosting and collaborative development. The UniTrack AI repository stores all frontend and backend source code. Also serves as the deployment trigger for Vercel (frontend auto-deploy on push) and Railway (backend continuous deployment).

System Analysis & Design
UML diagrams, system modeling, and documentation tools
Diagram / Tool Category Role & Purpose
🔷 ERD (Crow's Foot)
Data Modeling
 UML / DM Entity Relationship Diagram using Crow's Foot Notation documenting all database tables (users, roles, projects, tasks, notifications), primary/foreign keys, and 1:N cardinality rules. Essential for ensuring data integrity in the relational schema.
👤 Use Case Diagram
Requirements Analysis
 UML Captures all functional requirements and actor interactions with the system. Documents 8 use cases across 3 actors (Student, Supervisor, Admin) including Authentication, Task Upload, Grade Submission, User Management, and Report Generation.
📊 Activity Diagram
Workflow Modeling
 UML Models step-by-step process flows using swimlanes for each actor (Student, System, Supervisor). Documents the complete task submission workflow from upload through validation, supervisor review, approval/rejection decision, and final grading.
📨 Sequence Diagram
Interaction Modeling
 UML Illustrates time-ordered message exchanges between system components. The Task Submission sequence shows interactions between Student UI, API Controller, Task Service, Database, and Notification Service — making the system's runtime behavior explicit.
🔷 Class Diagram
OOP Structure
 UML Defines the object-oriented structure of the system: User base class with Student, Supervisor, and Admin subclasses (inheritance), plus Project and Task with their attributes and methods. Guides the backend model definitions.
🌊 DFD (Level 0 & 1)
Data Flow Analysis
 DFD Data Flow Diagrams document how data moves through the system. Level 0 (Context) shows the system as a single process with three external entities. Level 1 decomposes into four core processes: Authentication, Project Management, Task Submission, and Grading — each with their associated data stores and flows.
🖊️ Lucidchart / Draw.io
Diagramming Tool
 Design Tool Professional web-based diagramming tools used to create all UML, DFD, architecture, and flowchart diagrams in this documentation. Support UML 2.5 standard notation, export to SVG/PDF, and collaborative editing for team environments.
🏗️ 3-Tier Architecture
System Architecture
 Architecture Documents the three-tier system architecture: Presentation Tier (React.js browser client), Application Tier (FastAPI server on Railway), and Data Tier (SQLite database). Confirms clear separation of concerns and defines inter-tier communication protocols (REST API over HTTPS, SQLAlchemy ORM).
🚀 Deployment Diagram
Infrastructure Design
 Architecture Maps the physical deployment topology: Client Browser (React SPA) → Vercel CDN (static files, HTTPS) → Railway App Server (FastAPI + Python) → SQLite database file, with an external connection to the Google Gemini AI API.
